# Model Ladder

## Mini Low

- Use for repo inventory.
- Use for asset classification.
- Use for evidence capture and file organization notes.
- Do not use for implementation design or code rewrites.

## Medium

- Use for architecture/spec pass.
- Use for boundary decisions.
- Use for migration sequencing and interface contracts.
- Do not use it to start building the Godot client.

## Frontier

- Use for actual Godot implementation.
- Use for detailed UI construction, input flow, and client behavior.
- Use when code needs to be written, integrated, and debugged.
- Do not use it for first-pass repo cleanup or inventory work.

## Rule Of Thumb

- Mini Low for "what is here?".
- Medium for "what should it become?".
- Frontier for "build it now."
